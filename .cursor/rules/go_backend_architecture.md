# Go Backend Architecture - Structure and Layers

## Stack
Go 1.26.0 + MongoDB 8 + Swagger/OpenAPI

## Structure (DDD Layers)

```
/cmd/api/                    # Composition root and DI
/internal/{module}/
  /domain/                   # Entities, Value Objects, domain errors
  /app/                      # DTOs, Ports (interfaces), Services
  /adapters/                 # Repositories, MongoDB indexes, integrations
  /ports/                    # HTTP Handlers, Request/Response, Router
/pkg/{util}/                 # Agnostic utilities (no internal/, no .env)
/mocks/                      # Generated by mockery (do not edit)
```

## Dependency Flow
`ports → app → domain ← adapters`

## Imports with Aliases
```go
import (
    authapp "momento/internal/auth/app"
    authdomain "momento/internal/auth/domain"
    authadapters "momento/internal/auth/adapters"
    authports "momento/internal/auth/ports"
)
```

## Composition Root (cmd/api/main.go)

Required sequence:
1. Logger setup
2. Load config (`.env`)
3. Connect to MongoDB
4. Create indexes
5. Inject dependencies (adapters → app → ports)
6. Configure routes
7. Start server + graceful shutdown

See `cmd/api/main.go` as reference.

## pkg/ (Reusable Utilities)

Absolute rules:
- ❌ DO NOT import `internal/`
- ❌ DO NOT read environment variables
- ✅ Domain-agnostic (reusable across projects)
- ✅ Receive ALL configs via parameters

Examples: `pkg/mongodb/client.go`, `pkg/nethttp/nethttp.go`, `pkg/logger/logger.go`
